🛠 Шаблон: оплата в Telegram-боте через YooKassa В открытом канале @ai_do_now выложил концепцию и репо. Здесь - подробная инструкция по установке, разбор каждого шага и типичные грабли. Что получишь Рабочую систему приёма оплат в твоём Telegram-боте через YooKassa. Поддерживает карты МИР и любые другие. С чеками (ФЗ-54). С автоматическим уведомлением пользователю об оплате. Без Telegram Payments (в РФ заблокирован). Что нужно - n8n self-hosted или Cloud (есть бесплатный тариф) - Telegram-бот через @BotFather - Аккаунт YooKassa (kassa.yookassa.ru) - регистрация бесплатно, есть тестовая среда - Домен или белый IP для webhook'а (если self-hosted) - 30-40 минут на установку Шаги установки 1. Telegram-бот Открой @BotFather → /newbot → задай имя → получи токен типа 8616350126:AAGU...xxx. Сохрани этот токен надёжно - он даёт полный доступ к боту. 2. Доступы YooKassa Зайди в kassa.yookassa.ru → магазин → раздел «Интеграция» → «API ключи». Получи: - shopId (число типа 1388599) - secretKey (длинная строка типа live_AB1c...) Если только тестируешь - возьми тестовые ключи в разделе «Тестовый магазин». 3. Скачай шаблон github.com/Denis02861/telegram-yookassa-template В папке workflows/ два файла: - 01-bot-create-payment.json - сам бот (9 нод) - 02-yookassa-webhook.json - обработчик webhook от YK (6 нод) 4. Создай credentials в n8n Settings → Credentials → New: - Telegram API - вставь токен бота из шага 1 - HTTP Basic Auth - назови «YooKassa», в Username укажи shopId, в Password - secretKey 5. Импортируй workflow Workflows → Import from File → загрузи 01-bot-create-payment.json. Потом так же 02-yookassa-webhook.json. 6. Подставь credentials В обоих workflow открой каждую ноду где написано REPLACE (все Telegram-ноды + Create YK Payment). Выбери созданные credentials. 7. Правки в коде В 01-bot-create-payment.json → нода Build YK Payment: - return_url поменяй на ссылку своего бота: https://t.me/YOUR_BOT_USERNAME - В receipt.customer.email поставь свой fallback-домен - Тарифы в ноде Router подкрути под себя (объект TARIFFS) 8. Подключи webhook YK В n8n активируй 02-yookassa-webhook, открой ноду Webhook → скопируй Production URL (типа https://your-n8n.com/webhook/yookassa-webhook). В личном кабинете YooKassa → Интеграция → Уведомления HTTP → добавь этот URL. Подпишись на события: payment.succeeded (минимум), payment.canceled (опционально). 9. Активируй бота В n8n включи 01-bot-create-payment. С этого момента бот реагирует на /pay и callback'и. 10. Тест Напиши боту /pay → должно появиться меню тарифов. Нажми «Купить» → придёт ссылка на оплату. Оплати тестовой картой YK (номер 5555 5555 5555 4477, любой будущий срок, CVC 123). Должно прийти сообщение «Оплата прошла». Типичные грабли [-] Кнопка «Купить» нажимается, но ничего не происходит → проверь что credentials Telegram API подставлены во ВСЕХ нодах. Часто забывают про Send Tariffs [-] YK возвращает «Authentication failed» → перепроверь shopId и secretKey, в Username должен быть shopId без кавычек [-] YK возвращает «Idempotence-Key required» → проверь что в HTTP-ноде Create YK Payment в Headers есть Idempotence-Key (он формируется в Build YK Payment) [-] Платёж создаётся, но webhook не приходит → проверь что URL в личном кабинете YK совпадает с Production URL ноды Webhook, а не Test URL. Test URL работает только пока workflow открыт в редакторе [-] Webhook приходит, но пользователю не отправляется «Оплата прошла» → проверь что Telegram-нода Send to User в webhook-workflow подключена к нужному chat_id. По умолчанию chat_id берётся из metadata, проверь что в Build YK Payment metadata.chat_id заполнен Как добавить базу данных Сейчас шаблон работает без БД - вся информация о платеже передаётся через metadata. Если нужна история платежей и подписок: 1. Создай Data Table в n8n (или подключи свой Postgres) 2. В 01-bot-create-payment после Create YK Payment добавь ноду n8n Data Table → action: Insert. Поля: payment_id, chat_id, tier, amount, status='pending', created_at 3. В 02-yookassa-webhook после Parse Event добавь ноду n8n Data Table → action: Update by filter. Filter: payment_id, поля: status, paid_at Так можно отслеживать подписки, повторные оплаты, делать отчёты. Лицензия MIT. Используй, переделывай под свой продукт, продавай.
